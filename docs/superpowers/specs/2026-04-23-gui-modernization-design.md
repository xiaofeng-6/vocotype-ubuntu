# VocoType GUI 美化设计文档

## 目标

将现有 `tkinter` 桌面窗口从“灰底堆叠 + 视觉噪音偏高”的样式，升级为更现代、清爽、层级清晰的卡片式界面，同时保持现有录音功能逻辑不变。

## 已确认的设计决策

- 视觉风格：现代卡片风（清爽、轻阴影、弱边框）
- 信息密度：平衡
- 布局结构：顶部状态条 + 中央大录音按钮 + 次级说明 + 底部退出
- 状态文案仅保留两种：
  - `就绪`
  - `录音中`
- 删除“处理中（N 段）”相关状态展示

## 结构设计

### 1) 顶部状态区

- 左侧显示固定前缀 `状态` 与状态文本（`就绪` / `录音中`）
- 右侧显示状态指示点：
  - 就绪：灰蓝
  - 录音中：红色
- 状态区使用扁平圆角条样式，避免旧界面文字散落感

### 2) 主交互区

- 中央保留大圆形录音按钮，作为唯一主操作焦点
- 保留按钮三态反馈：默认 / 悬停 / 按下
- 录音中与非录音中使用不同主色系（红色系 vs 蓝色系），强化反馈

### 3) 辅助信息区

- 按钮下方显示动作提示：
  - 非录音：`点击开始录音`
  - 录音中：`点击停止录音`
- 快捷键提示保留常驻，但视觉降级为次级文本（低对比、小字号）

### 4) 底部操作区

- 右侧保留 `退出` 按钮
- 按钮使用次级样式，不与主录音按钮竞争视觉注意力

## 视觉规范（实现级）

- 窗口背景：浅灰蓝（接近 #f3f5fb）
- 主卡片：白底 + 12~16px 圆角 + 细边框 + 轻阴影
- 文本层级：
  - 标题/关键状态：深色（高对比）
  - 说明文字：中灰蓝（中对比）
  - 辅助文字：浅灰蓝（低对比）
- 控件间距更统一（8/12/16 的节奏），提升整洁度

## 行为与状态逻辑

- 保持 `toggle_recording(self._rt.worker)` 触发机制不变
- 状态刷新仍由轮询驱动（当前 `_refresh_recording_state` 机制）
- 状态计算简化为：
  - `is_running == True` -> `录音中`
  - 其余 -> `就绪`
- 不再读取并展示 `pending` 队列文案
- 异常处理保留：切换失败时 `messagebox.showerror(...)`
- 退出确认保留：录音中退出时二次确认

## 非目标（明确不做）

- 不改动录音核心、转写核心、热键监听核心
- 不引入第三方 GUI 库或主题库
- 不改动跨文件架构，仅在 `app/gui.py` 内重构界面表现层

## 测试与验收标准

1. 视觉验收
   - 主层级清晰：状态区、主按钮、辅助区、退出区清楚分层
   - 整体观感显著优于当前灰底版本，具备现代卡片感
2. 交互验收
   - 鼠标悬停/按下按钮反馈正常
   - 点击按钮可开始/停止录音
3. 状态验收
   - 非录音状态只显示 `就绪`
   - 录音状态只显示 `录音中`
4. 稳定性验收
   - 录音中关闭窗口仍有确认弹窗
   - 录音切换异常仍可弹出错误提示
